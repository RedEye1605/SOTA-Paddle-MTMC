# --- PP-Human / PaddleDetection MOT deps (PATCH 2026-06-18) ---
# PaddleDetection's MOT pipeline (JDE/FairMOT/ByteTrack
# linear assignment, JITed image ops, MTMCT evaluator) imports
# ``lap``, ``numba``, and ``motmetrics`` lazily inside the
# inference thread. If they are missing the subprocess
# prints its config banner then hangs on the first
# inference frame while the import raises — the parent
# sees no stdout for the entire ``PPHUMAN_STALL_TIMEOUT_SEC``
# window and the stall watchdog respawns the subprocess
# forever. Pinned here to make the dependency explicit
# in the runtime image rather than relying on
# Dockerfile-side ``pip install``.
